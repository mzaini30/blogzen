Mulai membuat aplikasi dengan Svelte

Ini adalah langkah-langkah membuat aplikasi dengan Svelte

Svelte adalah sebuah UI framework yang memiliki keunggulan dibandingkan tiga besar (React, Angular, Vue). Jikalau ketiganya menggunakan virtual DOM, Svelte nggak. Semua reactivity di Svelte adalah real DOM seperti jQuery. Tapi tentu saja berbeda dengan jQuery yang memiliki size sekitar 90 kb hanya untuk librarynya. Begitu pula dengan Vue yang memakan sekitar 90 kb untuk core frameworknya. Kalau di Svelte, 90 kb itu sudah jadi aplikasi yang lengkap. Bahkan, sudah memiliki banyak halaman.

Mengapa hal itu bisa terjadi dengan Svelte? Apa yang membuat keajaiban ini terjadi?

Itu karena berbeda dengan framework dan library yang terkenal itu (React, Angular, Vue). Kalau ketiganya itu, mereka mengincludekan core framework mereka ke dalam aplikasi yang kita buat. Kalau Svelte, dia nggak mengincludekan core frameworknya melainkan mengubah file-file Svelte dan file-file Javascript yang kita buat itu ke dalam bundle.js yang seoptimal mungkin dan memiliki ukuran yang sangat kecil.

Perbedaan lainnya yang membuatnya unggul adalah sintaksnya yang nggak memerlukan dummy DOM seperti di Vue. Kalau di Vue, dia penggunaannya seperti ini:

<template>
    <div v-for="x in hari">
        <div v-if="x == 'Minggu'">
            <p>Liburan...</p>
        </div>
        <div v-else>
            <p>Masuk kerja</p>
        </div>
    </div>
</template>

<script>
export default {

  name: 'NamaHari',

  data () {
    return {
        hari: ['Senin', 'Selasa', 'Rabu', 'Kamis', 'Jumat', 'Sabtu', 'Minggu']
    }
  }
}
</script>

<style lang="css" scoped>
    p {
        color: blue;
    }
</style>

Terlalu mubazir elemen div kan ya? Udah gitu juga, dalam penggunaan scriptnya cukup ketat karena semua berada di dalam object.

Nah, coba kita bandingkan dengan Svelte yang akan menghasilkan result yang sama:

<script type="text/javascript">
    const hari = ['Senin', 'Selasa', 'Rabu', 'Kamis', 'Jumat', 'Sabtu', 'Minggu']
</script>
<style type="text/css">
    p {
        color: blue;
    }
</style>
{#each hari as x}
    {#if x == 'Minggu'}
        <p>Liburan...</p>
    {:else}
        <p>Masuk kerja</p>
    {/if}
{/each}

Wow. Lihatlah. Itu bahkan seperti Javascript biasa aja, kayak nggak ada campur tangan suatu framework. Tentu hal ini sangat mustahil dilakukan jika Svelte bukan sebagai framework namun sebagai compiler. Maka, sampai ada yang mengatakan bahwa Svelte ini memang bukan framework Javascript bahkan ini adalah bahasa pemrograman yang baru. Tentu ini karena syntatic sugar yang ada di Svelte seperti each dan if kalau di contoh di atas.

Mulai menggunakan Svelte

Sekarang kita mulai menggunakan Svelte. Nah, sebelumnya, pastikan dulu kamu sudah menginstall Node JS dan terutama pnpm di laptop maupun di HPmu. Kalau di HP, berarti kita mainan Termux di lokasi folder ~. Kalau sudah, berarti kita coba mulai menggunakan Svelte.

Pertama, jalankan perintah berikut di Terminal/Termux:

pnpx degit sveltejs/template latihan-svelte

Lalu, kita tunggu beberapa saat sampai laptop mendownload atau mereuse template dari Svelte. Sebentar aja sih nunggunya sebenarnya.

Kemudian, kita masuk ke folder latihan-svelte:

cd latihan-svelte

Lalu, jalankan perintah berikut untuk menginstall semua dependencies dari Svelte. Ini nggak besar-besar juga sih total sizenya:

pnpm i

Lalu, kita buka di text editor kesayangan kita. Kalau aku sih, menggunakan Sublime yang aku buka dengan perintah berikut:

subl .

Membedah App.svelte

Nah, App.svelte adalah inti dari aplikasi kita. Kalau di HTML, bisa dibilang seperti index.html. Dan kalau di PHP, berarti index.php. Isi dari App.svelte saat pertama kali kita menggunakan template ini adalah:

<script>
    export let name;
</script>

<main>
    <h1>Hello {name}!</h1>
    <p>Visit the <a href="https://svelte.dev/tutorial">Svelte tutorial</a> to learn how to build Svelte apps.</p>
</main>

<style>
    main {
        text-align: center;
        padding: 1em;
        max-width: 240px;
        margin: 0 auto;
    }

    h1 {
        color: #ff3e00;
        text-transform: uppercase;
        font-size: 4em;
        font-weight: 100;
    }

    @media (min-width: 640px) {
        main {
            max-width: none;
        }
    }
</style>

Dari mana name?

Dia berasal dari main.js:

import App from './App.svelte';

const app = new App({
    target: document.body,
    props: {
        name: 'world'
    }
});

export default app;

Oh ya, App.svelte dan main.js itu berada di folder src. Nah, nanti kita olah-olahnya itu di folder src itu.

Semua isi dari folder build itu adalah hasil pengolahan Svelte. Lebih tepatnya adalah bundle.css dan bundle.js. Kalau file map dari kedua file itu adalah file keterangan struktur dari bundle.css dan bundle.js. Ya memang berasal dari pengolahan Svelte juga. Tapi kan, yang kita gunakan itu adalah bundle.css dan bundle.js. Kalau file mapnya itu hanya berisi keterangan struktur Javascript dan CSS dari file-file yang bersangkutan.

Lalu, yang tampil di browser itu adalah index.html. Ini isinya:

<!DOCTYPE html>

    <meta charset='utf-8'>
    <meta name='viewport' content='width=device-width,initial-scale=1'>

    <script defer src='/build/bundle.js'></script>

</body>

Nah, kita kotori tangan sedikit ya. Pertama, kita ubah dulu dari href='/global.css' menjadi href='global.css'. Begitu pula dengan yang lainnya.

Mengapa kita melakukan ini? Tentu supaya kalau kita menjalankan aplikasi ini bukan di root domain, dia masih bisa jalan. Misalnya aja seperti situs.com/latihan-svelte.

Lalu, kita ubah sedikit keterangan website ini supaya ramah kepada mesin pencari dan user tentunya:

Kembali lagi ke folder src

Sekarang kita kembali ke folder src dan hapus semua isi dari App.svelte.

Lalu, ketika kita buka browser dengan alamat localhost:3000 tadi, kini nggak ada isinya dan hanya tersisa tulisan Latihan Svelte di tabnya. Ya iyalah tentu isinya hilang. Kan kita hapus tadi. Kalau kita tekan Ctrl U, maka akan muncul isi dari index.html yang kita lihat tadi.

Kemudian, kita isi App.svelte dengan kode berikut ini:

<h3 class="tengah">List teman</h3>
<hr>
<p>Ini adalah list teman-temanku...</p>
<ul>
    <li>Satu</li>
    <li>Dua</li>
    <li>Tiga</li>
</ul>
<form>
    <input required type="" placeholder="Tambahkan teman" name="">
</form>
<style type="text/css">
    .tengah {
        text-align: center;
    }
</style>

Kalau dari kode di atas, nggak terjadi apa-apa ketika kita ingin menambahkan teman di kolom input, nggak terjadi apa-apa.

Oh iya, kalau mau lihat hasil dari kode di atas, bisa dibuka https://rawcdn.githack.com/mzaini30/latihan-svelte/a8a0cf38a128a5d7a470471144b714130a8a04de/index.html.

Nah, sekarang, kita kasih deh fitur dari Svelte supaya ini aplikasi menjadi dinamis dan nggak pakai loading kalau pas interaksi:

<h3 class="tengah">List teman</h3>
<hr>
<p>Ini adalah list teman-temanku...</p>
<ul>
    {#each list as x}
        <li>{x}</li>
    {/each}
</ul>
<form on:submit|preventDefault={tambahkan}>
    <input required type="" placeholder="Tambahkan teman" bind:value={baru} name="">
</form>
<style type="text/css">
    .tengah {
        text-align: center;
    }
</style>
<script type="text/javascript">
    let list = []
    let baru = ''
    const tambahkan = () => {
        list = [baru, ...list]
        baru = ''
    }
</script>

Hasilnya bisa dilihat di https://rawcdn.githack.com/mzaini30/latihan-svelte/b9809cfb80ced0b3576e6b56e09b21c40db400c6/index.html.

Kalau kita lihat dari hasilnya itu, dia kan jadi reactive gitu. Setiap ada perubahan data, dia akan terisi ke dalam list di atasnya.

Terakhir...

Terakhir adalah build. Untuk build, kita gunakan perintah berikut:

pnpm run build

Maka, file-file Svelte yang kita kerjakan tadi akan tersimpan di folder public/build dengan nama bundle.css dan bundle.js. Nah, kalau sudah, folder public inilah yang siap kita gunakan di hosting manapun. Bahkan, hosting yang nggak ada PHPnya pun bisa karena ini cuma HTML, CSS, dan JS biasa. Bahkan, ukurannya pun sangat kecil:

4.0K    bundle.css
8.0K    bundle.js

Amazing banget!

Untuk bundle.css bisa dilihat di https://rawcdn.githack.com/mzaini30/latihan-svelte/b9809cfb80ced0b3576e6b56e09b21c40db400c6/build/bundle.css. Sedangkan untuk bundle.js bisa dilihat di https://rawcdn.githack.com/mzaini30/latihan-svelte/b9809cfb80ced0b3576e6b56e09b21c40db400c6/build/bundle.js.
